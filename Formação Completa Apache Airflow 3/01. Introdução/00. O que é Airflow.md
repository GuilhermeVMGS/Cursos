Airflow é uma plataforma de orquestração de dados. Ela nos permite criar, agendar, executar e monitorar cada etapa de um pipeline através de uma DAG. Ou seja, ele coordena e processa diversas tarefas.

Embora seja para orquestração, ele é amplamente usado para transformação e carga de dados.
*Importante:* ele não processa em si, apenas orquestra workloads (principalmente batch)..
* Batch é processamento em lote. Como por exemplo, rodar um pipeline todo dia às 02h, atualizar o DW, processar vendas do dia anterior, etc.
* Embora sua principal forma de processamento seja em batch, ele possui uma característica de sensores e operadores "deferrable" que permite um processamento próximo ao tempo real.
* **Deferrable:** é uma característica que suspende o processo enquanto ele espera que um evento ocorre evitando ocupar o processo do airflow.

O processamento coordenado pelo airflow ocorre em:
* Sistema Operacional
* Banco de Dados
* Spark
* Elastic Search
* Etc.

O airflow possui interfaces WEB, REST API e CLI.
